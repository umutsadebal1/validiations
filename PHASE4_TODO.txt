🎯 PHASE 8 - DATABASE SCHEMA MIGRATION
────────────────────────────────────────────────────────────────

Mevcut tablo: HashHistory (5 sütun)

Yeni tablolar gerekli:

□ FileHashLog (Detaylı hash kaydedici)
  - Id (INT PK)
  - FilePath (TEXT UNIQUE)
  - FileHash (TEXT)
  - Algorithm (TEXT)
  - FileSizeBytes (BIGINT)
  - Status (TEXT: OK/CHANGED/ERROR/UNVERIFIED)
  - CreatedAt (DATETIME)
  - LastVerifiedAt (DATETIME)
  - VerifiedCount (INT)
  - Version (INT)
  - IsValid (BOOLEAN)
  - CreatedBy (TEXT)

□ FileHashHistory (Tarihsel versiyon takibi)
  - Id (INT PK)
  - FileHashLogId (INT FK)
  - OldHash (TEXT)
  - NewHash (TEXT)
  - ChangeReason (TEXT)
  - ChangedAt (DATETIME)
  - ChangedBy (TEXT)

□ VerificationLog (Doğrulama olayları)
  - Id (INT PK)
  - FileHashLogId (INT FK)
  - Status (TEXT: MATCH/MISMATCH/ERROR)
  - TimeTakenMs (INT)
  - VerifiedAt (DATETIME)
  - VerifiedBy (TEXT)
  - Notes (TEXT)

Migration tools:
  □ DatabaseService.MigrateToV2() metodu
  □ Backup önceki veriler
  □ Schema versioning (v1 → v2)
